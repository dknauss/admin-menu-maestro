  <action>
Create `tests/integration/AdminBarTest.php` following the LocalizationTest conventions
(`namespace Maestro\Tests\Integration; extends WP_UnitTestCase;` registered via the
`Maestro\Tests\Integration\` PSR-4 autoload already in composer.json).

DECISION (documented): VALIDATION.md lists `tests/unit/AdminBarTest.php`, but `Admin_Bar::node()`
depends on WordPress runtime (`is_admin()`, `current_user_can()`, `WP_Admin_Bar`, `add_query_arg`,
`is_edit_mode()`, `capability()`), and the unit suite (`tests/bootstrap-unit.php`) is deliberately
WP-free (loads only class-ordering + class-config). LocalizationTest only asserts the JS i18n payload,
which does NOT contain these admin-bar strings — so relying on it would leave UX-08b unverified.
Therefore create an INTEGRATION test (runs under `npm run test:php`, real WP) instead of a unit test.

Test design:
  - `set_up()`: `parent::set_up();` then
    `wp_set_current_user( self::factory()->user->create( array( 'role' => 'administrator' ) ) );`
    `set_current_screen( 'dashboard' );` (so `is_admin()` is true). `unset( $_GET['maestro_edit'] );`
  - Render the node by constructing a real `\WP_Admin_Bar`, calling `do_action( 'admin_bar_menu', $bar )`
    (the Admin_Bar constructor hooks `node` on `admin_bar_menu` prio 100 — confirm the plugin
    instantiates Admin_Bar during bootstrap; if not, instantiate `new \Maestro\Admin_Bar()` then fire
    the action, OR call `( new \Maestro\Admin_Bar() )->node( $bar )` directly). Then read back
    `$bar->get_node( 'maestro-toggle' )`.
  - test_enter_label: with NOT editing (no `maestro_edit`), assert the node `title` CONTAINS
    'Edit Menu' and the leading `dashicons-edit` span, and assert `meta['title']` contains
    'Edit Admin Menu' (the long accessible form).
  - test_exit_label: with `$_GET['maestro_edit'] = '1'` (editing), assert the node `title` CONTAINS
    'Exit' and `dashicons-exit`, and `meta['title']` contains 'Exit Editor'.
  - Use `assertStringContainsString` against the rendered title HTML. Reference the strings as the
    literal expected values (these are the UX-08b contract). Do NOT assert exact full-HTML equality
    (brittle against WP wrapping) — substring assertions only.
  - `tear_down()`: `unset( $_GET['maestro_edit'] ); parent::tear_down();`

IMPORTANT — Wave 0 state: this test is written against the TARGET strings ('Edit Menu'/'Exit'), so it
is EXPECTED TO FAIL against the current code ('Edit Admin Menu'/'Exit Editor' in the title). 11-02
makes it green. Add a class-level docblock noting it is the UX-08b guard and intentionally red until
11-02.
