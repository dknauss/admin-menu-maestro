<objective>
HARD-01 — scope `custom_menu_order` enablement to a stored top-level order. Replace
the unconditional `add_filter( 'custom_menu_order', '__return_true' )` at
`includes/class-replay.php:59` with a small Replay instance predicate that returns
`true` only when a non-empty `top_order` is stored, so Maestro stops claiming core's
menu-order machinery on every admin load when it has no opinion about ordering.

Purpose: interop correctness (defense-in-depth) — other plugins/themes that also hook
`custom_menu_order` / `menu_order` are no longer overridden when Maestro has nothing to
order. `reorder_top()` already no-ops on an empty `top_order` via `Ordering::top()`, so
this is an alignment, not a behavior change for users who HAVE set an order.

Output: a predicate-gated `custom_menu_order` callback in class-replay.php and
integration assertions in ReplayTest.php proving the gate (false on empty, true on a
stored order). Pure-PHP; no Docker required for the code change, integration assertions
run at the Wave 1 merge / wave boundary.
</objective>


New PRIVATE method (no WP dependencies — pure config read, cached in-request by Config):
```php
/**
 * `custom_menu_order` filter callback. Only claim core's menu-order machinery
 * when we actually have a stored top-level order; otherwise pass through so other
 * plugins/themes that hook custom_menu_order/menu_order are not overridden.
 *
 * @return bool
 */
private function has_top_order() {
    $cfg = $this->config->get();
    return ! empty( $cfg['top_order'] );
}
```

NOTE: register the predicate on the filter (not a one-time constructor check) — WP calls
custom_menu_order every admin load, so the callback must read config at call time.
The `menu_order` filter registration (reorder_top, line 60) stays UNCHANGED — it is
ignored by core when custom_menu_order returns false, and already no-ops on empty
top_order. Do NOT remove or gate it.

Existing reorder_top contract already proven safe (RESEARCH § Code Examples):
```php
public function reorder_top( $menu_order ) {
    $cfg     = $this->config->get();
    $desired = isset( $cfg['top_order'] ) ? $cfg['top_order'] : array();
    return Ordering::top( $desired, (array) $menu_order ); // Ordering::top(array(), $m) === $m
}
```
