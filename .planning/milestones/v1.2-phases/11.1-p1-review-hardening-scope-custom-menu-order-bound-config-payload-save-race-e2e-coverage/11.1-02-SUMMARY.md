## Decisions Made

- **Single GREEN commit for both tasks:** The plan specifies two separate commits for Task 1 (title + collections) and Task 2 (data-URI). In practice, all six constants and all cap enforcement were added to the same two files together. Splitting would require two partial states where some constants exist but not others — the pre-commit hook (running the full suite) would reject the intermediate Task 1-only commit because data-URI test assertions reference `Config::MAX_DATA_URI_BYTES`. Combined into one atomic commit.
- **WP stubs in bootstrap-unit.php (not the test file):** The test file used unbracketed `namespace Maestro\Tests\Unit;` which cannot be mixed with a bracketed `namespace { }` global block in the same file (PHP fatal error). Stubs moved to the bootstrap where they can be defined in the global namespace without mixing namespace styles.
- **hidden_roles pure-unit (no integration split):** The `wp_roles()` stub returns 60 named roles (role-1...role-60); `sanitize_key()` preserves them through; `array_intersect` passes all 51 roles through, and the cap slices to 50. Coverage complete without Docker or WP stack.
- **data-URI drop-not-truncate:** `substr()` on a base64 string at an arbitrary byte boundary produces a corrupt string. Over-limit icons are dropped to '' so the replay engine falls back to the existing WP menu icon cleanly.
