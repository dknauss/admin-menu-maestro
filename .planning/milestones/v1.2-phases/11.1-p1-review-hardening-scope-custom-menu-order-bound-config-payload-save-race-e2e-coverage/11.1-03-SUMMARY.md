## Files Created/Modified

## Decisions Made

- **Race (a) URL-based exit detection:** Asserting `maestro_edit=1` presence/absence rather than matching `D.exitUrl` avoids coupling the test to the server-computed exit URL, which varies by environment.
- **Race (b) request counter over negative timeout:** A `page.on('request')` counter is deterministic and cheap vs. a `Promise.race([waitForResponse, timeout])` which is timing-sensitive.
- **Race (c) response-order array:** Pushing `'POST'` and `'DELETE'` into a shared array inside `waitForResponse` callbacks records ordering without sleeps; `responses[0] === 'POST'` is the assertion.
- **E2E run deferred to Wave 2 boundary:** Docker/wp-env is unavailable in the GSD executor sandbox (same as Phase 9). The full Playwright suite runs sandbox-disabled at the Plan 04 boundary gate. HARD-03 is NOT marked green until that gate passes.
