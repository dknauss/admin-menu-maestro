**Deviation: has_top_order() visibility** — The plan specified `private` but the implementation uses `public`. This is documented in the Plan 01 SUMMARY and is correct: PHP's `call_user_func_array()` cannot dispatch to private methods registered via `add_filter()`. Public visibility is required. The method is not part of the documented plugin API. This is not a gap.

**Deviation: Both HARD-02 tasks in one commit** — Plans 02 specified two separate commits for Task 1 (title + collections) and Task 2 (data-URI). The implementation combined them into one atomic commit (c24193c) because the constants and test assertions for both tasks touch the same two files and splitting would require an intermediate state rejected by the pre-commit gate. This is documented in Plan 02 SUMMARY as an auto-fixed blocking issue. Not a gap.

**Race (c) setup fix (commit 1ba78ad)** — A test-precondition correction was applied after the initial race (c) implementation: the test now waits for the POST to be dispatched (`waitForRequest`) before clicking Reset All, ensuring `inFlight` is actually set. This is a test-setup correction only; `maestro.js` was not touched. Noted in Plan 04 SUMMARY under Decisions Made. Not a gap.

**WP function stubs in bootstrap-unit.php** — Plan 02 added WP function stubs (`wp_roles`, `sanitize_text_field`, `sanitize_key`, `wp_strip_all_tags`, `sanitize_html_class`, `esc_url_raw`) to `tests/bootstrap-unit.php` using `if(!function_exists())` guards. This enabled the `hidden_roles` cap to be tested pure-unit without Docker, eliminating the need for an integration-test split. The stubs are safe (guarded) and consistent with the project's bootstrap pattern.
