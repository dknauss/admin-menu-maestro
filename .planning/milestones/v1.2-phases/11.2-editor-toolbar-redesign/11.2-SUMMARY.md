# Phase 11.2 — Editor Toolbar Redesign (record)

**Goal:** The edit-mode toolbar is compact, coherent, and accessible — every
control is icon-only at all widths, the controls share one button system with
semantic colour, the status indicators read as indicators (not buttons), and the
toolbar earns its space (auto-clearing transient state, disabled controls when
there's nothing to do).

**Requirements:** UX-10 (Icon-only unified editor toolbar).

> **Provenance:** built via interactive design iteration (live wp-env + Playwright
> screenshots) rather than the standard `/gsd:plan-phase` → `/gsd:execute-phase`
> flow. It grew out of user feedback on the Phase 11 mobile toolbar work
> (UX-08/UX-08b). This SUMMARY is the retroactive record so the planning history
> matches what shipped. Inserted after Phase 11.1, before Phase 12, so Phase 12's
> release screenshots capture this final UI.

## What was built (8 commits)

1. `a8ed6bc` — panel buttons + mode/status indicators icon-only at **all** widths
   (was ≤600px only); visible label hidden, accessible name kept on `aria-label` +
   a `title` tooltip; the save-status word moved to an `aria-live` screen-reader-only
   span so it is still announced.
2. `a3c9521` — Reset All + Exit also icon-only (restore / × glyphs).
3. `7ae062a` — centred toolbar, uniform 20px glyphs, outlined square tiles for the
   mode/status indicators.
4. `ac7b9a2` — **unified control system**: one outlined box for every control + tile,
   semantic colour via `currentColor` (grey ordinary / green state / amber caution /
   red destructive); clearer Reset All glyph; lighter 1px zone dividers.
5. `6311c91` — vertically centre the in-button dashicons (reclaim WP core `.button`
   line-height).
6. `8a6c0b7` — gray-fill square-cornered buttons (gray hover; coloured glyph/border kept).
7. `36ef75b` — darkest gray fill (`#2c3338`) + match the rename field to the system.
8. `ab1c77d` — six behaviour/affordance refinements:
   - transient **"Saved" auto-clears** to idle after 2s (error stays);
   - mode/status are **flat glyphs** (no box) — visibly not clickable;
   - **Reset Item disables** (dimmed) when the item is unmodified; amber + enabled when
     there is a change to undo;
   - controls **grouped** (extra space before the Icon/Visibility/Reset-Item group);
   - **Exit → back-arrow** (`dashicons-arrow-left-alt`) instead of × (drops the
     "discard" read);
   - **palette icon-picker glyph** (`dashicons-art`) instead of the photo glyph;
   - **stronger destructive red** (`#ff5c5c`) on Reset All + error.

**Files:** `assets/maestro.js`, `assets/maestro.css`, `tests/e2e/editor.spec.ts`
(UX-03 mode-indicator test updated to assert the accessible name instead of visible text),
`package.json`.

## Accessibility

Icon-only ≠ inaccessible: every control keeps its `aria-label` (screen readers
announce "Move up", "Edit Mode", "Saved", etc.); sighted users get `title` tooltips;
the save-status word stays in an `aria-live` SR-only span; WCAG 2.5.5 tap targets
(≥44px at ≤782px) preserved; the destructive red meets non-text contrast on the dark
toolbar.

## Notes / follow-ups

- Icon-picker glyph landed as `dashicons-art` (palette); alternatives considered were
  `dashicons-screenoptions` (grid) and `dashicons-admin-appearance` (paintbrush).
- The earlier `flex-end` toolbar workaround (race(b) click-shift, shipped in Phase 11)
  was reverted to `align-items: center` here — icon-only compactness removed the panel
  wrap that `flex-end` guarded against, and the race(b) test is hardened test-side.
