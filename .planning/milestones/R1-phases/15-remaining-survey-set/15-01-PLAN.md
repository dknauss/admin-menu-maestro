<method_inheritance>

VERBATIM (adapting only plugin name/slug):
- **Hybrid source + runtime dump.** Read Jetpack's `admin_menu` registration code to explain HOW it
  manipulates the menu, then confirm WHAT happens via WP-CLI `$menu`/`$submenu` dumps.
- **`dump-menu.php`** (copy to SURV-02-assets, adapt header comment): hooks `admin_menu @ PHP_INT_MAX`
  (Maestro's replay priority), prints `pos⇥slug⇥title⇥icon⇥css` (top-level) and `pos⇥slug⇥title⇥cap`
  (submenu), exits before WP's per-user cap filtering. Run per role with `--user=admin|compat_editor|
  compat_shop_manager` and `--exec="define('WP_ADMIN', true);"`. **NOTE on WP_ADMIN:** WooCommerce
  needed it so `WC_Admin_Menus` (an `is_admin()`-gated class) would register; confirm at runtime whether
  Jetpack's menu requires it too (most plugins register on plain `admin_menu` regardless) and document
  the finding.
- **Dumps are REPLAY STATE, not the rendered sidebar** (SCHEMA convention 7): the dump exits before WP's
  render-time cap filter, so for non-admin roles it contains rows WP strips at render. Per-role Hide
  evidence MUST come from a separate rendered/post-cap-filter check (evaluate `current_user_can()` per
  row, or load the sidebar as that user) — never read "row present in dump" as "visible to role".
- **Top-level Reorder is classified from EFFECTIVE rendered order** via `reorder-probe.php` (copy/adapt),
  NOT the raw post-replay `$menu` global.
- **Config-driven op application:** write the sparse diff to `maestro_config`, re-dump, compare to the
  natural baseline, then `wp option delete maestro_config` between cases so they don't contaminate.
  Spot-check notable items in the real editor UI at planner discretion.
- **Per-role:** admin + compat_editor + compat_shop_manager (all provisioned). Jetpack ships no special
  role of its own, so the baseline three roles suffice — confirm at runtime.

  ```bash
  cd tests/compat
  npx wp-env run cli -- php -d memory_limit=512M /usr/local/bin/wp \
    --exec="define('WP_ADMIN', true);" \
    eval-file wp-content/plugins/maestro-menu-editor/.planning/compat/SURV-02-assets/dump-menu.php \
    --user=admin            # repeat for compat_editor / compat_shop_manager
  npx wp-env run cli wp option delete maestro_config           # natural baseline
  npx wp-env run cli -- wp option update maestro_config '<json>' --format=json   # apply an op
  ```
</method_inheritance>
