<objective>
Polish the edit-mode UI as a working admin tool (UX-02): clearer control
hierarchy and spacing in the bottom toolbar, unambiguous save/error status,
a more scannable ~20px icon-picker grid, and a one-time first-run cue — all
native to WP admin, with no text-overlap or control-resize regressions.

Purpose: The editor shipped functional but visually flat. The toolbar reads as an
undifferentiated row, status states lean on color, the icon grid looks ragged
(outline next to dashicons — Plan 01 fixes the weight; this plan tunes the grid
metric), and a first-time user gets no cue to "click an item to edit."

Output: Additive CSS in assets/maestro.css, minimal markup grouping + a dismissible
first-run cue in assets/maestro.js, and the cue's i18n strings in class-assets.php.
This is styling/glue — NOT unit-TDD'd (per the global skip rule); Plan 03 provides
the e2e regression guard and the before/after screenshot deliverables.
</objective>

<polish_scope_decision>
DECISION (Claude's Discretion #3 — scope bound): this is polish, not a redesign.
IN scope, prioritized:
1. **Control hierarchy + spacing** — group the toolbar into zones (status | selected
   label | edit controls | right-side reset/exit) with separators/gaps already
   hinted by `.maestro-panel { border-left }`; tighten alignment and gap rhythm.
2. **Save/error status clarity** — keep the existing color cues but add a
   non-color signal (a small leading glyph/dot or weight change) so saving/saved/
   error are distinguishable without color; ensure `.maestro-status` can't be
   overrun by a long selected-item label (it already has white-space:nowrap — verify
   no overlap when both are long).
3. **Icon-grid scanability at ~20px** — the grid is `minmax(36px,1fr)` cells with
   20px imgs; tune cell min-size/padding so solid glyphs sit on a consistent ~20px
   visual grid matching the dashicons tab; ensure the two tabs look like one system.
4. **First-run cue** — a small, dismissible inline hint ("Click a menu item to start
   editing") shown once, suppressed thereafter via localStorage. Native styling,
   keyboard-dismissible, aria-friendly.

OUT of scope: any layout re-architecture, new controls, color-token overhaul,
icon-upload, or anything touching REST/storage/selection behavior. Use existing WP
admin color tokens already in the CSS (#1d2327, #2271b1, #c3c4c7, etc.). Do NOT
collide with Phase 6's modified-indicator styles (06-03) — those classes
(e.g. .maestro-modified / per-item reset affordance) already exist in this file;
read them first and ADD around them, do not rewrite them.
</polish_scope_decision>
