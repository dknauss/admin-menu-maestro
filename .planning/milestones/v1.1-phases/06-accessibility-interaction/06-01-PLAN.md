<tdd_seam_decision>
DECISION (required by CONTEXT.md): **Option (a) — JS unit runner via `node:test`,
pure helpers extracted from `maestro.js`.** Justification:

1. **The logic is intrinsically frontend.** reorderMove operates on the live
   in-browser DOM order; diffItem compares the working `model` against the
   localized `pristine` payload; resetItem mutates the in-memory model and repaints
   immediately. Moving any of this to PHP would require new REST round-trips on
   every keystroke/move — which violates the milestone's "no new architecture / no
   REST contract changes" guardrail and would make the editor feel laggy.

2. **`node:test` adds ZERO runtime dependencies.** Node 24.16 (already installed)
   ships `node:test` and `node:assert` natively. No Vitest, no bundler, no new
   `devDependency`. `npm run test:js` = `node --test tests/js/`.

3. **Build hygiene is automatic.** `bin/build.sh` copies only `maestro.php`,
   `includes/`, `assets/`, `languages/`, `readme.txt` — it does NOT copy `tests/`.
   Putting the spec files under `tests/js/` keeps test tooling out of the runtime
   zip with no build-script change. `assets/maestro-logic.js` DOES ship (it is
   runtime logic), which is correct and intended.

4. **No build step in the browser.** `maestro-logic.js` uses a dual-export guard:
   `if (typeof module !== 'undefined' && module.exports) module.exports = api;`
   plus `if (typeof window !== 'undefined') window.maestroLogic = api;`. node:test
   imports it via `createRequire`/dynamic import of the CJS-style global; the
   browser reads `window.maestroLogic`. `class-assets.php` enqueues
   `maestro-logic.js` as a dependency of `maestro` so it loads first.
</tdd_seam_decision>
