Checker contract the executor MUST implement:

```js
// bin/check-doc-links.mjs
export const IN_SCOPE = ['README.md', 'SPEC.md', 'TESTING.md', 'docs/user-guide.md'];
// .planning/ prose is in DOC-01 scope but has near-zero real refs; include if a
// scan shows linkable offenders, otherwise IN_SCOPE above is sufficient. readme.txt is NEVER scanned.

export const CORE_EXCLUDE = new Set(['common.js', 'menu-header.php', 'wp-admin/menu-header.php']);

// Stale-path remaps: a ref token whose literal path is absent but maps to a real file.
// These are REPORTED as offenders (they must become a link to the corrected path) in plan 08-02.
export const STALE_REMAP = {
  'maestro.php': 'maestro-menu-editor.php',     // README repo-layout prose
  'icon.svg': '.wordpress-org/icon.svg',         // README repo-layout prose (bare ref)
  'global-setup.ts': 'tests/e2e/global-setup.ts' // TESTING / SPEC bare ref
};

// Offender = { file, line, token, resolvedPath, reason }
export function findOffenders(repoRoot = process.cwd()): Offender[];
```

Detection rules (implement exactly):
1. Token shape: a backtick-delimited inline-code span whose content matches a file path with an extension: `/^[A-Za-z0-9_./-]+\.[A-Za-z0-9]+$/`.
2. SKIP tokens inside fenced code blocks (```), inside an existing markdown link `[...](...)` (text or href), or inside an image `![...](...)`.
3. SKIP tokens in CORE_EXCLUDE.
4. Resolve: token relative to the linking doc's directory. If that file exists → candidate. Else if STALE_REMAP[token] exists and resolves → candidate (resolvedPath = remap).
5. A candidate is an OFFENDER (must be linked) only if it is NOT already wrapped in a markdown link.
6. readme.txt is never in IN_SCOPE.

CLI behavior: `node bin/check-doc-links.mjs` prints each offender as `file:line  \`token\` -> resolvedPath  (reason)`, then a count; exit 1 if any offenders, else exit 0. `--json` flag prints findOffenders() as JSON.
