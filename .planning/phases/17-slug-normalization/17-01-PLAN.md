  <name>Task 2: GREEN — implement Maestro\Slug::normalize() until SlugTest passes</name>
  <files>includes/class-slug.php</files>
  <behavior>
    All SlugTest cases from Task 1 turn green. The implementation realizes the
    mandatory pipeline (CONTEXT-locked order):
      (1) html_entity_decode FIRST (single full pass, ENT_QUOTES|ENT_HTML5),
          applied to the whole input — so `&amp;` becomes `&` before any `&` is
          read as a query separator. Non-string/empty → return ''.
      (2) Split off the fragment on the FIRST `#` only (preserve the rest verbatim).
      (3) Strip scheme+host: if input starts with the passed-in $admin_base,
          reduce to the admin-relative remainder (e.g. `admin.php?page=...`).
          For an internal wp-admin URL on a DIFFERENT host, detect the
          `/wp-admin/` boundary and keep everything after it (this is what makes
          a host move survive). For a genuine EXTERNAL URL (host not matching
          admin host, e.g. wpforms.com), keep `host/path` with the host
          LOWERCASED. Plain non-URL slugs pass through untouched.
      (4) Split the query on `&`; drop params whose NAME (case-insensitive) is
          `ver` or matches `utm_*`; keep all others including duplicates and
          empty-value params; do NOT lowercase param values, paths, or page=
          values. Sort surviving params alphabetically by the raw `name=value`
          token. Recompose `path?sorted_query#fragment`, omitting `?` when no
          params survive and `#` when no fragment.
    Idempotent: re-running on already-normalized output is a no-op (decode of an
    already-decoded string, host already stripped, params already sorted).
    Never throws.
  </behavior>
