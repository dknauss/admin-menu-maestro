<resolve_design>
<!-- The locked single-code-path design. Build a normalized lookup view, with
     a collision guard, once per replay() — then match rendered slugs by their
     normalized form. -->

1. Build a normalized lookup map for `$items` ONCE per replay():
   `$norm_items[ normalize( $stored_key, $base ) ] = $override` for every stored
   key. COLLISION GUARD: if two distinct stored keys normalize to the SAME key,
   that normalized key is AMBIGUOUS → mark it skip (do not apply any of the
   colliding overrides). Track a skip-set so an ambiguous key applies nothing.

2. At each items[] seam, look up by `normalize( $row[2], $base )`:
   - if the normalized key is in the skip-set → skip (apply nothing).
   - SECOND collision axis (CONTEXT): a normalized key that would apply to 2+
     DISTINCT rendered items in the same pass → also a no-op for that key. Detect
     by counting rendered matches per normalized key within the loop scope and
     skipping any key that matches more than one distinct rendered slug.
   - else apply the override exactly as today (title/icon/hidden_roles).

3. Reorder seam (:140): `$cfg['sub_order']` is keyed by PARENT slug and lists
   child slugs. Normalize the parent key when locating `$submenu[$parent]`
   (the parent itself may be an absolute/encoded slug), and pass a desired list
   such that `Ordering::submenu` compares NORMALIZED child slugs. Pick the
   mechanism (e.g. normalize both the row child slugs and the desired list inside
   a thin wrapper, or pre-normalize the desired list and compare against
   normalize($row[2])) — CONTEXT grants this threading choice. Keep
   Ordering::submenu's resilience contract intact (orphans skipped, newcomers
   appended).

4. NON-DESTRUCTIVE: never write normalized keys back into $cfg or the option.
   get_menu_model() (:246) is untouched — it keeps emitting RAW slugs to the JS.
   capture_pristine() is untouched.
</resolve_design>
