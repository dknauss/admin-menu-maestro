<objective>
UX-04 rename placeholder + accessible name: remove the visible "Rename " text label
that wraps the rename input and move the cue inside the field as a placeholder
("Menu label"), while preserving a programmatic accessible name via a visually-hidden
<label> so screen-reader and voice-control users still get "Rename".

Purpose: The visible "Rename " label eats horizontal toolbar space and is redundant
once the field is pre-filled with the item's title. A placeholder is cleaner — but a
placeholder is NOT an accessible name, so the SR label is mandatory, and the field
must stay pre-filled (placeholder only shows when empty). Rename commit/revert logic
does not change.

Output: new renamePlaceholder i18n key (+ LocalizationTest update in the same commit),
the rename field rebuilt in buildToolbar (SR label + placeholder, handlers intact),
::placeholder AA-contrast CSS, and e2e assertions. Depends on Plan 01 (placeholderVisible
seam) and Plan 02 (shares maestro.js/css/class-assets.php/LocalizationTest).
</objective>
