<maestro_internals>
<!-- The override surface the survey characterizes. Extracted from includes/class-replay.php + class-config.php. -->
- Maestro replays on `add_action('admin_menu', 'replay', PHP_INT_MAX)` — AFTER all other plugins (incl. WooCommerce) register. Rename/icon/visibility/submenu-order mutate the `$menu`/`$submenu` globals there.
- Top-level order goes through `custom_menu_order` + `menu_order` filters (run just after admin_menu), NOT by mutating $menu. `has_top_order()` only claims the machinery when stored `top_order` is non-empty.
- Stored config is a sparse-diff WP option named `maestro_config` (constant `MAESTRO_OPTION`). Shape:
  `{ "items": { "<slug>": { "title": "...", "icon": "dashicons-...", "hidden_roles": ["editor"] } }, "top_order": ["<slug>", ...], "sub_order": { "<parent_slug>": ["<slug>", ...] } }`
- Hide is per-role: an item with `hidden_roles` is cosmetically `unset()` from $menu/$submenu only for users whose roles intersect — direct-URL access still loads the page (cosmetic removal).
- Icon override drops core's `menu-icon-*` class for data/url icons so the custom image shows.
</maestro_internals>
