<objective>
Gap 1 (major, UX-08a): make the editor ENTER toggle reachable on mobile.

Root cause (UAT): includes/class-assets.php:55 `if ( ! is_edit_mode() ) return;` returns BEFORE
`wp_enqueue_style('maestro', …)` at :59, so assets/maestro.css — which holds the <=782px admin-bar
override at :512-521 — loads ONLY in edit mode. In the enter (non-edit) state the stylesheet is absent,
WP core's <=782px `display:none` for top-level admin-bar items wins, and the toggle disappears. The EXIT
toggle works only because edit mode loads the stylesheet.

Fix: extract the <=782px admin-bar override into a tiny new `assets/maestro-admin-bar.css` and enqueue it
UNCONDITIONALLY (above the early return), with a `dashicons` dependency so the pencil glyph renders. The
heavy editor `maestro.css` / `maestro.js` enqueues stay gated behind is_edit_mode() — non-edit pages gain
only ~1 small stylesheet, not the editor. Remove the now-duplicated override from maestro.css to keep a
single source of truth (maestro.css is owned by 11-07 this wave — see note).

Purpose: turn the 11-05 UX-08a enter-state guard GREEN.
Output: enter toggle visible + icon-only at <=782px on any capable-user admin page.
